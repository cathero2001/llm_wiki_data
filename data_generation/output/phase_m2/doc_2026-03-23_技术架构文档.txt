# WeBot 技术架构文档
版本：v1.0
作者：Dave Liu
日期：2026-03-23
状态：评审中

## 1. 整体架构

WeBot 采用 LangChain + 向量检索的 RAG 架构，整体分为三层：

```
┌─────────────────┐
│   用户界面层     │  Streamlit（MVP）/ React（v2）
├─────────────────┤
│   检索推理层     │  LangChain + 多后端向量检索
├─────────────────┤
│   数据存储层     │  向量数据库 + 文档存储
└─────────────────┘
```

## 2. 核心模块

### 2.1 文档处理管道
- PDF/Word/TXT 解析（pymupdf）
- 文本分块（RecursiveCharacterTextSplitter，chunk_size=500）
- 向量化（OpenAI embeddings）

### 2.2 向量检索层
- 接口统一封装（BaseVectorStore）
- 支持后端：Pinecone / FAISS / Milvus
- 查询流程：query → embed → top-k → rerank → 结果

### 2.3 对话管理层
- ChatHistory 持久化（SQLite）
- 多轮对话上下文（window=5）
- 会话隔离（per-user vector store namespace）

## 3. 向量数据库选型

| 方案 | 成本 | 运维 | 私有化 | 适用场景 |
|------|------|------|--------|----------|
| Pinecone | 高（年费10万） | 托管 | 否 | 快速验证 |
| FAISS | 免费 | 中等 | 是 | 成本优先 |
| Milvus | 免费 | 较高 | 是 | 规模化生产 |

当前：评估中，M3 给出 FAISS POC 结果后决定

## 4. 部署架构

```
                    ┌─────────────┐
                    │  用户浏览器 │
                    └──────┬──────┘
                           │ HTTP
                    ┌──────▼──────┐
                    │  Streamlit  │  (或 Nginx + React)
                    │   服务器    │
                    └──────┬──────┘
                           │ API
              ┌────────────┼────────────┐
              │            │            │
       ┌──────▼─────┐ ┌─────▼─────┐ ┌───▼────┐
       │  LangChain │ │  OpenAI   │ │ 向量库  │
       │   服务     │ │   API     │ │ Milvus  │
       └───────────┘ └───────────┘ └────────┘
```

## 5. 安全设计

- 用户认证：JWT Token（暂不考虑SSO，M1范围外）
- 数据隔离：per-namespace 向量空间
- 敏感信息：日志脱敏处理

## 6. 性能指标

- 查询延迟：P95 < 2s（M4目标）
- 并发支持：20 QPS（M1目标）
- 准确率：> 85%（RAG retrieval accuracy）
